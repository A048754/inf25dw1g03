## C1. Descripción general del proyecto

**Maia Campus Event** es una aplicación web compuesta por una API REST y un cliente de backoffice,
orientada a la gestión de eventos en el Campus Universitario de Maia.

El objetivo principal del sistema es facilitar la organización de eventos académicos y formativos,
evitando conflictos en la reserva de salas y centralizando la gestión de usuarios, eventos y recursos
asociados.

Este proyecto es una continuación del trabajo desarrollado en el Momento 2 (M2), manteniendo el
mismo dominio del problema, pero adaptándose a los nuevos requisitos del Momento 3 (M3), utilizando
una aproximación *code-first* con la framework LoopBack 4 y un cliente de backoffice desarrollado
con React-Admin.

La API sigue una arquitectura REST y permite realizar operaciones CRUD (Crear, Leer, Actualizar y
Eliminar) sobre los distintos recursos del sistema, utilizando los verbos HTTP GET, POST, PUT y
DELETE. El backoffice proporciona una interfaz gráfica para la gestión visual de dichos recursos.
