## C2. Recursos del sistema

La API REST está compuesta por los siguientes recursos principales:

### Users
Representa a los usuarios del campus que interactúan con el sistema.
Cada usuario dispone de un identificador único, nombre, correo electrónico institucional
y un tipo de usuario (estudiante, profesor o administrador).

### Rooms
Gestiona la información de las salas disponibles en el campus.
Cada sala cuenta con un identificador único, nombre, edificio y capacidad máxima.

### Events
Almacena los eventos que se realizan en el campus.
Cada evento tiene un identificador único, título, fecha y hora de inicio y fin, y está asociado
a una sala concreta.  
Un evento es creado por un usuario del sistema.

### Bookings
Representa las reservas asociadas a los eventos.
Cada reserva identifica un evento concreto y permite gestionar la ocupación del mismo.

### Tags
Permite clasificar los eventos por tipo o temática (por ejemplo: programación, talleres, charlas).

## Relaciones entre recursos

- **Users → Events**  
  Relación 1:N.  
  Un usuario puede crear varios eventos, pero cada evento es creado por un único usuario.

- **Rooms → Events**  
  Relación 1:N.  
  Una sala puede albergar múltiples eventos en distintos momentos.

- **Events → Bookings**  
  Relación 1:N.  
  Un evento puede tener varias reservas asociadas.

- **Events ↔ Tags**  
  Relación 1:N.  
  Un evento solo puede tener una etiqueta y una etiqueta puede estar asociada a varios eventos.
